# LINE 官方帳號預約管理系統 (LINE Reservation System)

這是一個專為 LINE 官方帳號量身打造的強大預約管理系統，全站採用獨特的**「手繪漫畫風 (Neo-brutalism)」**介面設計，色彩鮮明且極具互動感。系統結合了 LIFF (LINE Front-end Framework) 與現代化的 React 前端技術，不僅提供客戶無縫的預約體驗，更賦予管理者全方位的後台控制權限、自動排班演算法與資料視覺化介面。

---

## 🌟 系統特色與詳細功能說明

### 1. 📱 客戶端功能 (LIFF 預約介面)
客戶無需另外下載 App 或註冊帳號，直接從 LINE 官方帳號點擊選單連結，即可自動登入並開始預約。

*   **填寫基本資訊**：預約前要求輸入「孩子姓名」(或預約者姓名)，確保管理者能清楚辨識參與者。
*   **智慧型項目選擇**：
    *   系統會動態顯示開放中的「預約項目」(例如：秋季體驗課、一對一諮詢)。
    *   **防呆與限制機制**：
        *   **次數限制**：若該項目規定每人只能預約 1 次，當使用者預約達上限時會跳出提醒。
        *   **重複預約鎖定**：若使用者在該項目已經有「已核准」的預約，按鈕將自動反灰並顯示「已獲核准，無法重複預約」，避免佔用其他名額。
        *   **黑名單/權限控制**：若管理者在後台限制某位使用者，該使用者將無法點擊該預約項目。
*   **直覺的日期與時段選擇**：提供直覺的月曆介面，自動鎖定不開放或已額滿的日期；點擊日期後即時顯示剩餘時段與名額。
*   **我的近期預約 (歷史紀錄)**：
    *   使用者可隨時查看過去與未來的預約紀錄。
    *   若預約狀態為「待審核」，使用者可以自行點擊垃圾桶圖示取消預約。
    *   若預約已被管理者「已確認」，系統會自動鎖定刪除功能，防止誤刪已成立的資料。

---

### 2. 🛡️ 管理者後台功能 (Admin Dashboard)
後台採用極具特色的漫畫風格設計，包含粗黑邊框、鮮豔填色與立體陰影，讓管理工作也能充滿樂趣。後台涵蓋以下核心模組：

#### 🧩 預約時段安排 (最終排班神器)
這是系統的核心亮點功能，專門解決「多人搶同時段」或「一人預約多個志願」的排班衝突問題：
*   **一鍵自動安排 (Auto-Scheduling Algorithm)**：
    *   系統內建回溯排班演算法 (Backtracking)。只需點擊「✨ 一鍵自動安排」，系統會自動為所有待審核的預約者尋找最佳的時段配置，確保每個人都能排入且時段不衝突。
    *   **多重方案選擇**：若演算法計算出有多種排班組合（例如有 3 種排法都能滿足所有人），系統會彈出選項視窗，讓管理者預覽並選擇最喜歡的排班方案。
*   **視覺化拖曳排班 (Drag & Drop)**：
    *   所有的時段會以網格呈現。如果有兩個人同時預約同一個時段，該時段會亮起**紅底**並顯示「衝突！請拖曳移開」。
    *   管理者可以直接用滑鼠拖曳使用者的卡片，將其移動到其他空閒的時段，直到所有衝突解除（綠底最佳狀態）。
*   **發布最終安排**：
    *   當排班確認無誤後，點擊發布，系統會**自動執行以下批次動作**：
        1.  將格子內的所有人狀態轉為「已核准」。
        2.  **落選處理**：自動將該時段落選的其他待審核人員取消。
        3.  **多志願處理**：若被核准的人同時預約了其他時段當作備案，系統會自動取消他其他的待審核時段，釋出名額。
        4.  **自動推播**：透過 LINE API，瞬間發送「預約成功確認」推播訊息給所有獲選的使用者。

#### 📅 預約管理 (Reservations)
*   **狀態分流**：清楚區分「待審核」與「已核准」的預約清單。
*   **月曆總覽**：上方月曆即時顯示每天的預約狀況。支援「台灣繁體節氣與國定假日」顯示，並以紅底標示週末，讓排程更直覺。
*   **單筆彈性操作**：可手動單獨核准/拒絕預約、將已核准退回待審核、或「重新傳送確認」推播給特定客戶。
*   **資料匯出**：支援將預約資料匯出為 Excel 格式，方便線下建檔。

#### ⚙️ 預約設定 (Availability)
*   **項目字典管理**：可自訂多個預約項目，獨立設定名稱、開放期限、每人預約次數上限，甚至設定「限制預約名單」。
*   **彈性的排班系統**：選定特定日期與時段，指派開放的預約項目與人數上限。
*   **快速複製功能**：可將某一天的排班設定，一鍵複製到同月份的其他指定星期 (如：一鍵套用到所有的週二與週四)，大幅節省建立時段的時間。

#### 👥 用戶管理 (Users)
*   **客戶資料庫**：自動記錄曾使用過系統的 LINE 用戶，並賦予獨立序號。
*   **進階欄位與篩選**：
    *   支援設定「群組」、「性別」、「生日」(系統自動換算星座)、「興趣」、「自訂標籤」(如 VIP、黑名單) 與「內部備註」。
    *   擁有強大的多維度過濾器，可快速篩選出特定條件的客群。
*   **自訂名稱鎖定**：管理員可修改使用者的顯示名稱 (例如改為小孩名字)，修改後系統將鎖定該名稱，不受使用者更改 LINE 暱稱影響。
*   **一對一專屬推播**：在列表點擊訊息圖示，即可直接發送客製化的 LINE 文字推播給該名使用者。

#### 💬 訊息設定 (Messages)
*   **客製化推播訊息 (Flex Message)**：
    *   自訂客戶「送出預約時」與「管理員最終核准時」收到的 LINE Flex Message 圖文訊息標題與內容。
    *   支援上傳自訂的背景底圖 (支援 JPG/PNG)，讓推播更具備品牌特色。
*   **動態變數支援**：訊息內文支援使用 `{好友的顯示名稱}` 等變數。
*   **原始名稱優先切換**：可勾選「優先使用原本的 LINE 名稱」，確保即使管理員將名稱改為小孩姓名，推播打招呼時仍可使用家長的 LINE 暱稱。

#### 🛠️ 系統設定 (Settings)
*   支援在前端介面直接設定 LINE Channel Access Token，不需修改環境變數即可動態切換綁定的官方帳號。

## 💻 技術架構

- **前端框架**: [React 18](https://react.dev/) + [Vite](https://vitejs.dev/)
- **資料庫與後端**: [Firebase (Firestore, Storage)](https://firebase.google.com/) + Vercel Serverless Functions (`/api/send-line-message`)
- **樣式設計**: [Tailwind CSS](https://tailwindcss.com/) (全站採用 Neo-brutalism 漫畫風設計語言)
- **圖示庫**: [Lucide React](https://lucide.dev/)
- **日期與農曆運算**: [date-fns](https://date-fns.org/) + [lunar-javascript](https://github.com/6tail/lunar-javascript)
- **資料匯出**: [SheetJS (xlsx)](https://sheetjs.com/)

## 🚀 本地開發與運行

1. **安裝依賴套件**
   ```bash
   npm install
   ```

2. **設定環境變數**
   請參考 `.env.example`，建立 `.env` 檔案並填入您的 Firebase 與 LINE API 金鑰。

3. **啟動開發伺服器**
   ```bash
   npm run dev
   ```

4. **編譯打包 (供 Vercel 或其他靜態伺服器部署)**
   ```bash
   npm run build
   ```
